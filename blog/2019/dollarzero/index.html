$0 - Do you use it ?

Published: 19 Feb 2019

What $0 does.

I am wondering how many people actually know what $0 does, not a lot of developers I talk to know about it - let alone use it. so lets dive into it.

$0 equals your selected element

Select something with either the element selector or by clicking an element in the element tab of the developer tools. You will notice a small '$0' next to your selected element (pictured above). If you enter "$0" into the console it will print out your selected element.

I often use it when testing out code - it is a lot quicker to develop something in the browser as opposed to an slow CMS that potentially is caching old changes etc.

/* selecting an element with the ID "item" */
$0.id // will return "item"
$0 == document.getElementById("item") // same same
$($0) // will return the jquery representation of the element
      // i.e. $("#item")

/* Note the true effectiveness of the $0 feature is when the element is not unique, i.e. classess*/
/* Say there are 10 elements on the page that share the class "randomClass" class 
    By selecting one of them with the selector we can target our code easily */

$0.classname == "randomClass" // true
   // $0 has a direct reference to the instance of the element by class
   // say we selected the 4th one 
$0 == document.getElementsByClassName("randomClass")[3] // could be true who knows

   // My point being - it is super helpfull to experiment with your code
   // I find my self useing it a lot when I have to rely on DOM structure crawling

$1 equals your PREVIOUS element.

Treat it like a stack of selected elements that pops upwards everytime you select a new element.
